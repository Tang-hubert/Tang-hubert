- 其他 Cron Jobs（Moltbook、i-En、Hack Skill、Session Monitor）均無路徑依賴，不受影響

**Phase 2：Memory 歸檔機制**

### memory/archive/ 不佔 Context 的原因
- OpenClaw workspace 注入基於 AGENTS.md 明確規則（非目錄掃描）
- archive/ 不在規則內，等同「沉睡區」，零自動讀取負擔
- 只有 Memory Maintenance heartbeat 或人為主動叫用才會接觸

### Orchestration Framework 檢討
- 確認過去專案（Constitution、pronhub-hotspot-clipper）均未真正使用 orchestration
- 原則確立：**所有專案預設走 orchestration framework**，除非 Hubert 明確說不要
- opencode 定位：外包特殊任務（非取代 sessions_spawn）

### MiniMax Runtime 工具缺口
- apply_patch、memory_search、memory_get 三工具 MiniMax 不支援
- 不影響目前所有日常專案開發
- 只在 subagent 派遣時出現警告，無功能影響
