### AI Agent Building Philosophy (From YouTube)

#### Core Philosophy: Simple & Cruel
- **API Call > Agent**: If one API call can solve it, don't use Agent
- **Workflow > Agent**: If steps are fixed/deterministic, use Workflow
- **When to use Agent**: When task needs human involvement (aesthetic judgment, iterative refinement)

#### Skills/Tools
- Agent fails → lack of tools, not bad prompts
- **Emergence**: When you give Agent core tools, they combine 1+1>2
- Start with simple SDK, not LangGraph

#### Prompting Strategy
- Don't copy-paste big company prompts (Token爆炸)
- Start with zero constraints
- Iterate: add constraints based on AI behavior

#### Context Engineering
- **Context pollution**: Too many tools/history = performance drop
- **Planner-Executor architecture**:
  - Planner: Has global info, dispatches tasks
  - Executor: Focuses on specific task
- Precision filtering: Only show necessary context

#### Memory Management
- **Pass by reference, not data**: Don't let AI repeatedly read/write large data
- **Internal Memory**: Only for current conversation
- **External Storage**: Persistent info across sessions (To-do lists, state tracking)

#### Debugging & Audit
- Record **process**, not just results
- Audit metrics: tool call order, token consumption, redundant context
- Review execution path to find optimization space

#### Summary: Run first, then optimize
- Don't let "elegant architecture" become a blocker
- Start simple, add tools/context/memory only when complexity demands it
