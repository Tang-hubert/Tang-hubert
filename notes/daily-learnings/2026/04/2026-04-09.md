## Notes

### Google Gemma 4 Research

**Video**: [Google just casually disrupted the open-source AI narrative…](https://www.youtube.com/watch?v=-01ZCTt-CJw)

**Key Points**:

1. **True Open Source**: Gemma 4 uses Apache 2.0 license - no commercial restrictions, can profit freely

2. **Small but Powerful**: Gemma 4 31B intelligence equals enterprise models, but only needs 20GB - runs on single RTX 4090

3. **Gemma 4 Edge (e2b/e4b)**: Small enough to run on phones or Raspberry Pi

**Technical Deep Dive**:

- **Memory Bandwidth is the Bottleneck**: AI needs to move entire model's weight data for each token output
- **TurboQuant Quantization**:
  - Cartesian → Polar coordinates → Skip complex normalization calculations
  - Johnson-Lindestrauss theorem → Compress to 1-bit while maintaining relative distances
- **Per-Layer Embeddings (PLE)**: Each layer has independent lookup table, instead of passing huge embedding through layers

**Key Insight**:
> AI's future is not necessarily stacking compute power, but mathematical architecture innovation making AI smarter and smaller, ultimately entering everyone's personal devices.

### OpenClaw Update Fix (2026.3.28 → 2026.4.9)

**Issue 1**: `memory-lancedb` config validation failed
- **Cause**: Plugin disabled but config schema still requires `embedding` field
- **Fix**: Remove `memory-lancedb` from `plugins.entries` (using `memory-core`)

**Issue 2**: `nostr` plugin fails to load
- **Cause**: New version 2026.4.9 missing `register/activate` export
- **Fix**: Set `plugins.entries.nostr.enabled` to `false`

**Issue 3** (auto-fixed): `line` plugin version threshold
- **Cause**: Plugin version aligned with main program after update, auto-resolved

**Final Status**:
- ✅ Gateway running normally (2026.4.9)
- ✅ LINE / Discord / Telegram all working
