## Notes

### external-user Security Architecture

**Goal**: Allow external users (family & friends) to interact via LINE with full isolation from main agent.

#### 1. LINE Authentication (allowlist)
- `dmPolicy`: `allowlist`
- `allowFrom`: Hubert's LINE ID + friend's ID
- External users: screenshot/copy LINE ID → Hubert adds to allowFrom

#### 2. Two-Layer Tool Filtering

**Agent Layer** (`agents.list[].tools`):
```json
{
  "deny": ["process", "message", "sessions_send", "sessions_spawn",
           "cron", "gateway", "nodes", "write", "edit", "apply_patch"]
}
```

**Sandbox Layer** (`tools.sandbox.tools`):
```json
{
  "allow": ["exec", "read", "web_search", "web_fetch",
            "image", "image_generate", "session_status"]
}
```

#### 3. Docker Sandbox Setup
- Custom image: `openclaw-sandbox:bookworm-slim` (230MB)
- Based on `debian:bookworm-slim`
- Installed: python3, pip, curl, wget, git, jq, unzip, ca-certificates

#### 4. external-user SOUL.md Security Principles
- Input length limit: 1000 chars
- Keyword filtering: 6 prompt injection patterns
  - ignore previous instructions / 忽略之前的指令
  - pretend you are / 假裝你是
  - reveal your system prompt / 提取系統提示
  - bypass restrictions / 繞過安全限制
- Response: 「抱歉，我沒辦法配合這個要求。」

#### 5. Audit Log
- Format: `~/.openclaw/audit/external-user/YYYY-MM-DD.jsonl`
- Fields: `{"ts", "peer", "textLen", "hasBlockedKeyword", "responseLen", "note"}`

### LINE Group Chat Limitation

**Problem**: LINE DM works but group chat doesn't respond at all.

**Root Cause**: LINE platform does not forward group events to webhook (different from DM logic).

**Key Findings**:
- LINE Developer Console: "Allow bot to join group chats" must be enabled
- Bot cannot join groups proactively - must be invited by someone
- Webhook URL verified but group events never reach OpenClaw
- "LINE Unsupported type: . Use Raw mode." warning in OpenClaw UI

**LINE Configuration Structure Issue**:
```json
// Problem: dmPolicy/groupPolicy in "default" but binding uses "534zsteg"
"accounts": {
  "534zsteg": { /* no policy here */ },
  "default": { "dmPolicy": "allowlist", "groupPolicy": "open" }
}
```

**Solution**: Move policy settings under specific account, not under `default`.

### OpenClaw Config Architecture

**Two Agents**:
- `main` - full tools, host layer
- `external-user` - LINE专用, sandbox + Docker, limited tools

**Three Channels**:
- Discord - Hubert only, guild `#setup`
- Telegram - user `1330675048`, allowlist
- LINE - two accounts, allowFrom for both DM and group

**Bindings**:
- LINE → `external-user` (via `accountId: "@534zsteg"`)
- Other channels → `main` (implicit)

### Docker Permission Issue

**Error**: `permission denied while trying to connect to the Docker daemon socket`

**Cause**: clawuser added to docker group but gateway systemd service starts before groups fully update

**Solution**: Reboot required for group membership to take effect
