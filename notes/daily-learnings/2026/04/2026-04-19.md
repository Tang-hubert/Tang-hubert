## Notes

### Cognee MiniMax LLM Integration Breakthrough
- **Problem**: `LLM_PROVIDER=anthropic` + `ANTHROPIC_BASE_URL` went through wrong adapter (LiteLLM couldn't handle)
- **Solution**: `LLM_PROVIDER=custom` + `LLM_MODEL=openai/MiniMax-M2.7` + `LLM_ENDPOINT=https://api.minimax.io/v1`
- Architecture: `Cognee → GenericAPIAdapter → LiteLLM → MiniMax API`
- Verification: search response ✅ (1.67s / 4.99s)

### Ollama Embedding (replacing MiniMax)
- **Problem**: MiniMax embedding returns `vectors` format (not OpenAI's `data`), LiteLLM can't parse
- **Solution**: `EMBEDDING_PROVIDER=ollama` + `OLLAMA_EMBEDDING_MODEL=Qwen/Qwen3-Embedding-0.6B` + `EMBEDDING_DIMENSIONS=1024`
- Model pulled (639MB), `embed_text()` test 1024 dims ✅

### Cognee Vector DB Switch: LanceDB → PGVector
- **Reason**: LanceDB persistence bug (`add()` reports success but collection is empty, 0 rows)
- **Vector dimension incompatibility**: Old collection 3072-dim (MegaBite), Ollama is 1024-dim, deletion failed
- **PGVector Docker** started at `:5432`, psycopg2-binary installed

### Cognee Four "Limitations" Clarified
| Feature | Status | Explanation |
|---------|--------|-------------|
| Raw text input | ✅ SDK works | REST API requires file upload, but SDK `cognee.add("text")` supports raw |
| Pure vector search (no LLM) | ✅ CHUNKS works | `SearchType.CHUNKS` bypasses LLM |
| Fast original text retrieval | ✅ CHUNKS + verbose | Returns full chunk list with original text |
| Concurrent write | ❌ Kuzu lock | Real architectural limitation |
