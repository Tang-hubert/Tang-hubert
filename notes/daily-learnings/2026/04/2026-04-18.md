## Notes

### Skill Registry System
- **痛點**：舊檔案執行/移轉/清理太模糊；下載時不確定能做什麼；不確定系統是否有使用
- **架構**：
  ```
  ~/.openclaw/skills-registry/
  ├── bin/skill-registry.sh
  ├── manifests/
  ├── audit.log
  └── tests/
  ```
- **Skill vs Plugin 差異**：Skill 主動叫用，Plugin 被動全域自動觸發（auto-recall/autoIndex）
- **Manifest 過時解決**：安裝時強制產生 + 每小時 audit cron job 抓幽靈檔案
- **Bug 修復**：
  1. `jq -s` 改為 `jq -R '.' | jq -s '.'`（分兩步 pipe）
  2. JSON heredoc → `jq -n --arg` 構造避免 invalid JSON
  3. `grep ... || true` 避免 set -e 中斷
  4. `local name=$(...)` → `skill_name=$(...)`（非 function scope）
  5. `overlaps` 是 jq 內建關鍵字 → `.["overlaps"]` bracket notation
- **測試結果**：test_registry_json.sh 97項全通過，test_audit.sh 通過，test_cognee.sh 12項全通過

### Cognee-openclaw Plugin 部署
- **部署方式**：pip install（Docker 記憶體不足被 SIGKILL）
- **Python venv**：3.12（cognee 要求 `<3.14, >=3.10`）
- **DB 錯誤**：`COGNEE_DATABASE_PATH="$HOME/.cognee/databases"`
- **正確端點**：`cognee.api.client:app`（非 `cognee.api.main`）
- **systemd service**：`~/.config/systemd/user/cognee-server.service`

### Cognee MiniMax LLM 整合
- **問題**：Cognee 原生不支援 minimax，只支援 anthropic
- **解法**：使用 `LLM_PROVIDER=anthropic` + `ANTHROPIC_BASE_URL=https://api.minimax.io/anthropic`
- **Patch**：`AnthropicAdapter.__init__` 加入 base_url 參數
- **驗證**：`COGNEE_SKIP_CONNECTION_TEST=true`（MiniMax API 30s timeout）
- **結果**：`auto-sync complete: 0 added, 0 updated, 0 deleted, 55 unchanged` ✅
