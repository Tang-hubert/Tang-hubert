## Notes

### Claude Code 源碼洩漏研究

**事件背景**
- 2026-03-31 Claude Code 源碼意外透過 npm source map 洩漏
- GitHub repo `instructkr/claw-code` 在 2 小時內突破 50K stars（史上最快）
- Anthropic 緊急下架，但否認打算起訴

**研究結論**
1. **Agent Harness 定義**：LLM = 大腦，Harness = 身體（工具系統+記憶+協調+安全）
2. **Claude Code vs OpenClaw vs OpenCode 定位不同**：
   - OpenCode：輕量 CLI，純 coding
   - Claude Code：專業 IDE 等級 coding agent
   - OpenClaw：通用 AI 助手框架（跨工具整合）
3. **Hubert 的使用方式是合理的**：需要跨工具 workflow，OpenClaw 架構更順

**各工具定位**
- OpenCode：快速小型 coding 任務，多模型切換
- Claude Code：大型/封閉式 coding 任務，深度檔案狀態追蹤
- OpenClaw：跨工具整合，arbitrary 工作流

**可借鑒的設計（非底層）**
- Plugin 系統 → SKILL.md 結構化
- Tool Orchestration → AGENTS.md 流程定義
- 自我修正 loop → 失敗重試策略
- Context 壓縮 → Memory 整理習慣

**Context 滿前的自動總結**
- 觸發時機：session context > 55%
- 做法：濃縮重點寫入今日 memory，Session 內只留高層摘要
- 不需要 cron job，在 session 內主動判斷處理

**工具/技能失敗時的替代方案回報**
- 失敗立即回報 + 說明替代方案 + 爭取確認
- 保持輕量：一句話失敗、一句話對策

**跨 Session 任務連續性（.task-state/）**
- 位置：`~/.openclaw/workspace/.task-state/`
- 結構：`active-tasks.md` + `[project-name]/` 資料夾
- 使用時機：複雜任務開始時自動建立、完成後移除
- 觸發條件：Hubert 說「繼續」or 任務明顯複雜
