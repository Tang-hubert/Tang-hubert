## Notes

### Daily Summary Cron Job Backup Logic Fix

**Problem**: Hubert discovered that 04-13 cron backed up all state files (SOUL/USER/MEMORY/IDENTITY/TOOLS/HEARTBEAT), but none of these files actually changed.

**Root Cause**: Backup logic was comparing "yesterday's backup with today", but 04-12 backup doesn't exist, causing agent to conservatively back up everything.

**Fix - Step 5 Backup Logic Changed to mtime Comparison**:
- Use `stat` or `ls -l --time-style=+%Y-%m-%d` to check state file mtime (last modification time)
- mtime is today → backup to `backup/{filename}_YYYY-MM-DD.md`
- mtime is not today → skip, no write operations

**Benefit**: No longer need "yesterday's backup" as baseline, completely solves the problem of all files being re-backed up when backup files are not continuous.

### Git Sync Scope (unchanged)
- `memory/` (within 3 days)
- `daily-summary/`
- `backup/` (state files with changes that day)
