### SQLite Knowledge

**Benefits**:
| Dimension | Description |
|-----------|-------------|
| Storage | Single `.db` file, structured storage, higher compression than JSON/md |
| Query Efficiency | Indexed fields O(1); JSON/md requires full scan O(n) |
| Concurrency | Auto lock on multi-process write, won't overwrite like JSON |
| SQL Query | One line for filter/sort/stats; text files need grep |

- **Dangerous keywords**: Block `rm -rf /`, `sudo su`, `nc -e`, `curl | sh`, `wget | sh`, `;rm`, privilege escalation
- **Docker sandbox**: Network isolated, memory 512m, PID limit 64, non-root execution

### Phase 0 Supplement Scan Mechanism

**Problem**: Phase 5 commits each card sequentially (`git add → commit → push`). If timeout occurs mid-process, last cards don't get pushed.

**Solution**: Add Step 2 to Phase 0 - scan and push all uncommitted files

**Flow**:
1. `git pull origin main` sync
2. Scan `git diff --name-only HEAD` for uncommitted files
3. Commit by type:
   - `output/*.md` → `research: {card_name}—{first line title}`
   - `dashboard/research_progress.md` → `dashboard: update research_progress`
   - `dashboard/session_summary.md` → `dashboard: update session_summary`
   - Others → `chore: {filename}`
4. Creds scan before each push
