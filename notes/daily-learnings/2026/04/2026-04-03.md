## Notes

### Work-in-Midnight: Dispatcher-Worker Architecture

**Problem**: Isolated sessions can still run out of context when processing multiple cards.

**Solution**: New architecture with clear separation:
- **Dispatcher** (cron job): Lightweight coordinator that reads Pending cards and spawns workers
- **Worker** (per-card isolated session): Fresh context per card, 20-min timeout, outputs `[WORKER_RESULT]`

| Aspect | Benefit |
|--------|---------|
| Context | Fresh per card, no accumulation |
| Single card failure | Isolated, doesn't affect others |
| Timeout | 20 min per card independently |
| Complex cards | Don't slow down simple cards |

### Work-in-Midnight: File Naming Convention

**Correction**: Using `{date}_{slug}.md` causes continuity issues when continuing research on different dates.

**Fixed Design**:
- Research files: `{slug}.md` (no date prefix, same file appends across sessions)
- Session summary: `{date}_session_summary.md`
- Each research session separated by `## 第 N 次研究 — YYYY-MM-DD HH:MM`

### Work-in-Midnight: Continuity Model

**Key Insight**: Trello is progress visualization, `.md` is source of truth.

| Intent | Trello Action | Effect |
|--------|---------------|--------|
| Continue deeper research | Card stays in `complete` | cron job ignores |
| Trigger new round | Move card from `complete` → `Pending` | cron job detects, finds existing `.md`, continues from "待續..." |

**Recommended**: Just update the "待續..." section in `.md` — cron job auto-picks up next execution, no Trello manipulation needed.

### OpenClaw Workspace: Modular System Prompt Architecture

**Restructured** `~/.openclaw/workspace/` into modular components:

| File | Size | Content |
|------|------|---------|
| `SOUL.md` | - | Identity core |
| `USER.md` | - | User data |
| `AGENTS.md` | 3.4KB | Structure carrier (reduced from ~10KB, -66%) |
| `SKILLS.md` | 2.8KB | CSO Trigger Index (20 OpenClaw Skills) |
| `WORKERS.md` | 3.2KB | Smart Worker Discovery + Secrets + Task Definition |
| `RULES.md` | 4.9KB | Coding Mode + Context Rot + Safety + Heartbeats |

**Loading Principle**:
- SOUL + USER + AGENTS: Every session
- SKILLS / WORKERS / RULES: Lazy load on demand
