### IMAGE Tool Functionality (Discord Image Analysis)

**Process:**
1. **Discord upload** → Discord uploads image to CDN or receives via OpenClaw webhook
2. **OpenClaw storage** → Image saved to local path:
   ```
   /home/clawuser/.openclaw/media/inbound/[filename].png
   ```
3. **Message with path** → Message's `media` block indicates local file path
4. **`image` tool analysis** → Uses MiniMax vision model to analyze path

**image tool specs:**
- **Purpose**: Analyze image content (OCR + visual understanding + object/character recognition)
- **Input**: Image file path or URL
- **Model**: MiniMax vision model (integrated in `minimax/MiniMax-M2.7`)
- **Usage**: Must know file path first (from message's media block)

**Limitations:**
- Need actual file path to read
- Cannot directly read Discord CDN URL (must download to local)
